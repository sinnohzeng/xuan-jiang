# 声音匹配（voice matching）：让改稿贴作者的声音，而非贴“标准好文”

## 为什么要有这一层

去 AI 味的默认终点是“平实克制的好文”，但对随笔与自媒体，“好”还不够，还得读着像**这个作者**写的。一个反直觉的坑：把 AI 味抹干净后，文章常常也把作者原有的口头禅、句子节奏、偏爱的比喻一起抹平了，变成“正确但没有人”的稿子。constitution 把这个状态叫**无声**（仅 G6/G7 判失败）。

关键认知，反复申明：**不像 AI ≠ 像我**。去掉黑词和套路只是必要条件，不是充分条件。声音匹配这一层专门补“像我”这半程，且只在个人表达体裁里补；公文里“像我”是跑偏。

## 门控：只在 G6/G7 开

- **G6 随笔杂文 / G7 自媒体**：启用。个人声音是这两类的价值本体。
- **G1 规范性公文 / G2 领导讲话稿 / G3 调研报告 / G4 述职报告 / G5 汇报稿发言稿 / G8 第三方咨询报告**：关闭。这些体裁的“声音”是体裁规范（法定格式、机关口径、咨询体例），不是个人声纹；在这里追求“像我”等于破坏规范。

Polish 主对话在判定体裁后决定是否进入本层；不属 G6/G7 直接跳过，不提示、不追问。

## tier-1：即时观察（默认，不落盘）

用户主动要求“写得像我 / 贴我的风格 / 声音匹配”，且体裁为 G6/G7 时启用。

1. **取样**：请用户贴 2 至 3 段其**旧作真迹**（非本次待改稿）。不足两段则如实说明样本太少、只能弱匹配，不脑补作者风格。
2. **六维观察**（只观察真迹里**反复出现**的习惯，单次偶发不算）：
   - 句子节奏：长句短句的配比、有没有惯用的短句收尾
   - 用词偏好：口语还是书面、有没有反复出现的自选词或口头禅
   - 标点习惯：省略号、问号、感叹号的使用密度（仍守 GB/T 15834，习惯指密度不是违规标点）
   - 段落组织：习惯开门见山还是先叙后议、段落长短
   - 修辞倾向：爱用比喻、反问、引用，还是白描直陈
   - 语气与人称：克制还是热烈、第一人称密度、跟读者的距离
3. **生效**：把观察到的习惯作为本次改稿的**附加约束**并入 Polish step 3 的有序 sweep（不新增一遍 reviewer，声音只是改稿时的一层偏好）。改完在复盘里显式说明为贴合作者声音保留或恢复了什么。
4. **边界**：声音匹配**不能覆盖红线**。作者惯用“赋能”“闭环”也照删，声音是节奏和偏好，不是给黑词开的后门。作者惯用错别字/病句照改。

tier-1 全程不写任何文件，观察随对话结束即弃。

## tier-2：落盘声音画像（可选，用户项目仓）

用户希望以后每次都按其声音改时，可把画像落盘复用。

- **落到哪**：用户**自己项目仓**的 `.writing-voice/<genre>.md`（如 `.writing-voice/g6-essay.md`），genre-scoped 一体裁一份。**绝不**写进插件目录；插件不携带任何用户声纹，声纹是用户资产、留在用户仓。
- **画像内容**：上面六维的自然语言小结，加 2 至 3 条 before→after 的贴声音改写示例。**不存**用户原文全文、不做向量化、不做嵌入检索；这是一份人可读、可手改、可删的 md，不是黑箱声纹库。
- **复用**：**仍需用户在本次 Polish 明确要求声音匹配**（门控不变，不因画像文件存在就自动激活）。确认要求后，到用户项目仓根 `.writing-voice/<genre>.md` 检测画像是否存在，存在则读它作 tier-1 六维观察的替代输入（省去重新贴旧作），仍只作改稿附加约束。
- **失效**：用户改文件即改声音，删文件即关闭。零迁移、零后台状态。

（tier-3 语料库级声音学习 / 自动声纹提取推迟，无当前需求依据，不预建。）

## 非目标与红线

- **不为骗过 AI 检测器**：目标是文章真的像作者本人、真的好，不是让某个检测器把它判成“人写的”。
- **不引向量检索 / 嵌入库**：声音画像是可读 md，保持人可审可改可删，不做黑箱相似度。
- **不越体裁**：G1-G5、G8 一律不谈作者个人声音。
- **不覆盖任何红线**：标点、黑词、戏剧化隐喻、编造事实全部照常执行，声音层无豁免权。
