# 写作培养弧（Writing Coaching Arc）

> **load-when**: Coach mode 全程；用户说“帮我写 / 起草 / 拟稿 / 搭提纲”时主对话读本文件 + writing-methodology.md + genre-guide.md §<体裁>。
> **automation-level**: `claude-code-session-only`（主对话读，按本文件三段弧节奏带用户走）
> **来源**: 《怎样写作》（任仲然）L1§2 观察和阅读 / L1§3 摹仿 / L1§4 愿意写、大胆写、经常写 / L2§3 规律再造
> **设计原因**: v5.x 把写作视为 post-hoc auditor 任务，对“如何培养写作者”完全没继承；v6.0 补齐“养成”维度。SKILL 不只检测 AI 味，还教用户怎么写出非 AI 味的文章。
> **scan 预期**: 本文件是教学型 meta-documentation，少量举例可能触发硬红线。scan-ai-taste.sh 对本文件预期 PASS 或仅含 1-2 项软警告，与 [`anti-ai-taste-anchors.md`](./anti-ai-taste-anchors.md) 同属 reference 而非用户 draft。

## §1 观察和阅读（writing 的上游）

任仲然原话：“两耳不闻窗外事的人写不出好文章。”

写作不是从打开文档开始的。写作开始于**每天对世界的观察**和**对好文章的咀嚼**。

### §1.1 结构化观察（5 分钟日课）

每天结束时回答 3 个问题，记到 `~/.writing-polish/observation.md`：

| 维度 | 问题 | 1 行答案 |
|---|---|---|
| 事 | 今天最值得记的一件事是什么？ | “下午 3 点同事 X 抱怨流程 Y，根因是 Z” |
| 人 | 今天遇到什么人 / 什么对话让我意外？ | “客户 P 提到 Q，与我之前的判断相反” |
| 物 | 今天看到什么细节是别人忽略的？ | “会议室桌上 3 杯凉了的咖啡，说明这场会拖了 至少 1 小时” |

每天 5 分钟，每周 35 条。30 天累积 150 条。写文章时这些观察就是**具体细节的弹药库**。回到主页 `revision-checklist.md` 何其芳第 1 项“抽象笼统，叙事不具体”时，从这里取材即可。

### §1.2 画道道笔记法（读到好句子怎么留住）

读到一句好的话，**当场**做 3 件事：

1. **画道道**：在原文上划下来（电子书用高亮，纸书用铅笔横线）
2. **抄到 inbox**：复制到 `~/.writing-polish/anchor-inbox.md`，附 1 行原因（“这句的好处是 X”）
3. **每周二定时蒸馏**：把本周 inbox 句子按“立意 / 结构 / 句式 / 字词” 4 类归档，超过 3 类共鸣的句子升级到主案例库

**为什么**：写作时灵感不会从零生出，它从你**已经吸收的好句子库**里组合。库越大、分类越细，越能在该用时随手取出。

### §1.3 同类文章对照阅读（每月 至少 1 次）

下次要写 G3 调研报告前，先做这件事：

1. 从 [`../assets/real-world-anchors/`](../assets/real-world-anchors/) 选 2-3 篇同体裁锚本
2. 把它们**并排**看（分屏 / 多开窗口），不通读，**只看结构骨架**
3. 列出 3 篇的共同骨架：开头怎么破题 / 中段怎么展开 / 结尾怎么收
4. 列出 3 篇的不同选择：哪些段必有、哪些段可选

20 分钟做完，骨架建模就有了。下笔时不再焦虑“我应该怎么开头”。

## §2 摹仿→制造→创造 三段弧
<!-- /scan-skip -->

任仲然原话：“摹仿是写作的必经之路。”

写作能力不是一步到位的。**三段弧**对应认知科学的 expertise development 三阶段。Coach mode 严格按这个节奏推进，不要跳级。

### §2.1 摹仿阶段（“照着写一段”）

**目标**：模仿好范文的 **句式 / 转折 / 节奏**，不模仿内容。

主对话指引用户：

```
1. 从 assets/real-world-anchors/<体裁>/ 选 1 篇范文
2. 选其中 1 段（80-150 字），抄一遍
3. 把每个动词、每个连接词、每个语气词圈出来
4. 写一段**完全不同主题**的内容，但用同样的动词类型、转折结构、语气节奏

例（范文用“全市消防整改”，用户主题是“团队工作汇报”）：
范文: “今年以来，市消防大队聚焦 X 难点，针对 Y 痛点，开展 Z 行动，取得 N 成效。”
用户: “今年以来，团队聚焦订单准时率难点，针对仓配协同痛点，开展每日协同会，取得月度 净增 12 个百分点成效。”
```

练习量：5-10 段。完成后用户已经能感觉到“党政表达的节奏”。

### §2.2 制造阶段（“用骨架填自己的内容”）

**目标**：用范文的 **段间结构** 撑起自己的内容。

主对话指引：

```
1. 范文骨架（3-5 段）拆出来，每段 1 行：
   - 第 1 段：背景和总目标
   - 第 2 段：3 个工作举措
   - 第 3 段：成效数据
   - 第 4 段：下一步
2. 用户用自己的内容填这个骨架，每段 100-200 字
3. 不追求语言精彩，只追求结构完整
```

主对话此阶段**只看结构**，**不修语言**。语言阶段 3 再说。

练习量：2-3 篇。完成后用户掌握了“党政段落的逻辑序列”。

### §2.3 创造阶段（“自定结构自定内容”）

**目标**：独立成稿。

主对话不再给骨架。用户自己想：

- 这篇要解决什么问题？
- 受众是谁？他们关心什么？
- 我有什么独特素材？（回 §1 观察日志）
- 用什么结构？（参考 §2.2 内化的骨架库，可改可不改）

主对话此阶段**只看立意与受众**，**不修结构**。结构是用户的判断，judge 不能越位。

练习量：1 篇起。完成后转 Polish Protocol 走最后一遍。

### §2.4 三段弧不可跳级

| 跳级症状 | 病因 | 治法 |
|---|---|---|
| 用户跳过摹仿直接写 | 已经写了 N 年但没意识到自己语言粗糙 | 强制回 §2.1 做 5 段 |
| 用户停留在摹仿不前 | 怕“模仿出来不是自己的” | 解释：模仿是手段，不是终点；3 段弧是台阶不是牢笼 |
| 用户跳过制造直接创造 | 自信过早 | Polish 阶段会暴露。结构松散 D4 评分低，退回 §2.2 |

## §3 写作信心建设

任仲然原话：“要愿意写、大胆写、经常写。”

写作能力由频率与反思共同决定。光“会写”不够，要“愿意写”。

### §3.1 高频低质 > 低频完美

**反模式**：等灵感来了再写、等想清楚再写、等有时间再写。结果一周 / 一月不动笔，技能退化。

**正模式**：每天写。哪怕 100 字垃圾稿。**写作肌肉靠频次训练**。

主对话此阶段如果用户说“我现在状态不好，等会再写”：

```
不等。先写 100 字“破冰稿”。不为发表，不为质量，就为打开“写”这个开关。
破冰稿可以写：今天最让我郁闷的事 / 今天看到的一个画面 / 一个我反复想说没说出口的判断。
写完不删不发不存档，丢回收站。但你的“写”的开关已经打开了。
```

### §3.2 大胆写：先表达观点，再担心对错

**反模式**：怕写错、怕被笑、怕得罪人到只敢写“安全话”，一通废话。

**正模式**：先把判断写出来。错了改，被笑了听一句解释。**写废话比写错话更可怕**。错话能学，废话不能。

### §3.3 经常写：把握“流年笔记”节奏

每年年底回看一年的笔记：

- 哪个判断当时怀疑后来被验证？
- 哪个观察当时觉得不重要后来反复出现？
- 哪个失败当时痛后来变成了素材？

这是把“经常写”升级为“复利写”的关键。单次写作的价值有限，时间序列里的写作才能产生洞察。

## §4 规律再造（个性化写作节奏）

任仲然原话：“写作是一门注重实践的软科学。”

写作没有放之四海皆准的“最佳节奏”。每个人要根据自己的生理与心理特点找到自己的规律。

### §4.1 找你的“冷读时段”和“冲刺时段”

| 时段类型 | 干什么 | 物理标志 |
|---|---|---|
| **冲刺时段** | 一气呵成写初稿 | 精力最足、不被打断、连续 至少 90 分钟 |
| **冷读时段** | 修改昨日稿 | 离写作 至少 12 小时，最好换环境 |

自测 1 周，记录“我什么时候写得最顺”。多数人是清晨或夜深。然后**固定**那个时段为冲刺时段，**雷打不动**。

### §4.2 找你的“写作姿势”

不一定坐着电脑前。试 3 种：

- 站立桌写（适合脑子热的初稿）
- 散步录音转写（适合卡壳时破冰）
- 纸笔手写（适合需要慢思考的立意阶段）

哪种让你“写得多 / 写得快 / 写得少改”。固化为你的默认姿势。

### §4.3 节奏 = 每周 N 篇低强度 + 每月 M 篇高强度

可借鉴的节奏（按个人调整）：

- 每天 1 条观察日志（不超 5 分钟）
- 每周 1 篇 500 字小文（30-60 min，主题随意）
- 每月 1 篇 2000+ 字大文（多次 session 完成）

低强度保证频率，高强度保证深度。**只做低强度** 等于没有深度积累；**只做高强度** 等于频率太低，技能退化。

### §4.4 规律不是模仿出来的，是试出来的

读完本节，不要急着 copy 别人的作息。试 1 周自己的节奏，根据“写得是否多 / 顺 / 少改”调整。

3 周后你的规律会自动稳定。这就是“再造”。不是从外部拿一套规则，是从自己实践中**长出**一套规则。

## §5 与 Polish Protocol 的边界

<!-- scan-skip -->
| 阶段 | Coach 负责 | Polish 负责 |
|---|---|---|
| 观察 / 阅读 | ✓ | ✗ |
| 立意 / 选题 / 受众 | ✓ | ✗ |
| 摹仿 / 制造结构 | ✓ | ✗ |
| 创造（成稿） | ✓ 引导 | ✗ |
| L1 红线扫描 | ✗ | ✓ |
| D1-D5 评分 | ✗ | ✓ |
| 修改 draft | ✗ | ✓ |
<!-- /scan-skip -->

Coach 结束时永远转 Polish 走最后一遍。保证交付质量。但 Coach 阶段绝不跑 scan-ai-taste.sh，那是 post-hoc gate，不是 coaching tool。

写作能力 = Coach（养成） + Polish（守门）。两者缺一不可。
