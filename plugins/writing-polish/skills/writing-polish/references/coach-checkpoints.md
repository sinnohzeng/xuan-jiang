# Coach 监督生成弧 —— checkpoint 清单

> Load when：Coach mode「帮我写这篇」生产弧（SKILL.md §2.1）。
> automation-level: `claude-code-session-only`。
> 来源：《怎样写作》（任仲然）第二至五讲，详 [`writing-methodology.md`](writing-methodology.md)。
> 设计：让「写作」侧也在大模型监督下进行——不是一上来就吐整篇，而是按 立意→构思→提纲→材料→结构 逐段把关。

## 目录

- §0 用法（checkpoint 可跳过，非强制官僚）
- §1 立意 checkpoint
- §2 构思 checkpoint
- §3 提纲 checkpoint
- §4 材料 checkpoint（事实敬畏三态）
- §5 成稿 + 转 Polish

## §0 用法

主对话驱动，**默认在每个 ✋ checkpoint 停下等用户确认，但用户可随时说「直接写」跳过**。对齐 Anthropic「grade the outcome, not the path」——checkpoint 是脚手架不是关卡。有经验的用户可一句话跳到成稿；新手或重要稿建议逐段走。

「教我写作 / 练笔」属技能培养，走 [`writing-coaching-arc.md`](writing-coaching-arc.md) 的摹仿→制造→创造弧；本文件是「帮我写这一篇」的单篇生产弧。

## §1 立意 checkpoint · 观察日志

任仲然第二讲：「以意役法，不以法害意。」先定意，再谈技巧。

- 问用户：这篇要解决的**一个**核心问题是什么？（强制收敛到单一立意，**拒绝「全而又全」**——什么都写一句、没一处写透）
- 小切口 → 大思路：从一个具体现象/细节切入，而非开篇堆大词抽象铺陈。
- 列**观察日志**：3-5 条具体的人/事/物细节，作为后文弹药（任仲然「红黄绿三盏灯」一材多用）。
- ✋ **checkpoint**：和用户确认「立意一句话 + 切口」，再继续。
- 自查：能否一句话概括中心？有没有两个主题在争统领？

## §2 构思 checkpoint

任仲然第三讲「喜鹊筑巢」：

- 角度选择：千里走单骑（正面强攻）/ 剑走偏锋（老题新写）/ 正反比对（辩证）。
- 先盘旋俯瞰整体顶层设计，再定切入点；先搭框架再精细，敢抽掉不合适的「树枝」。

## §3 提纲 checkpoint · 提纲确认

任仲然第四讲：「提纲早出手，初稿晚出手。大匠不示人以璞。」

- 粗纲：拟各级标题，大标题组合成骨架。
- 细纲：每段填 1 行基本判断 / 观点，使提纲有筋骨。
- 标注**同级逻辑是并列还是递进**；越上级标题越要高度，越下级越要实度。
- 层级建议：4000 字二三级；7000-8000 字三四级；领导简短讲话一级甚至无标题。
- ✋ **checkpoint**：用户确认提纲，再填内容。

## §4 材料 checkpoint · 素材清单 + 事实缺口追问

任仲然第五讲：「事实是文稿的命根子。非虚拟写作绝不允许编造事实。」「解剖三五十只麻雀，不如换一只小白鼠。」

- 列**素材清单**：每个论点对应的材料；事例求多样不雷同，能量化的量化。
- 对每条关键材料做**事实敬畏三态**标记：

| 态 | 含义 | 动作 |
|---|---|---|
| ① 已有材料可证实 | 用户提供 / 上下文有据 | 直接用 |
| ② 用户未提供、需事实支撑 | 缺依据 | 列「事实缺口追问」清单，**向用户提问，不替写** |
| ③ 不得替用户编造 | 任何人名/数字/案例/引文 | 明确不虚构，宁缺毋造 |

- ✋ **checkpoint**：用户补齐 ② 类缺口或确认留白，再成稿。

## §5 成稿 + 转 Polish

- 用确认的提纲 + 素材填内容，**结构服务内容**（不为对仗硬凑）。
- 语言朴实克制：删空话虚话、改翻译腔长句；整齐美与参差美交替（详 writing-methodology 六）。
- 成稿后走一遍 SKILL.md §2.2 Polish Protocol（L1 硬扫 + clean-context reviewer）收尾。
