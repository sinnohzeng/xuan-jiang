# 去 AI 味离线锚本库（Anti-AI-Taste Anchors）

> 本文件是"去 AI 味"工作的**离线锚本资产**。下次处理泛体制内党政公文或咨询报告类文稿时，直接 Read 本文件即可取得正反对照样本，**无需每次重新用 Firecrawl 调研**。

## 为什么需要这份锚本

AI 生成内容在泛体制内文风上默认表现不佳，根本原因有三：

1. **训练数据盲区**：真实的国办文件、国家智库咨询报告、财政部工作通知在训练语料中密度极低，AI 默认生成的是"训练数据里最常见的公约数"——一种"看起来公文但读起来滑溜"的 AI 综艺体
2. **负面清单不够用**：只告诉 AI "不要用破折号""不要用如果说…那么…"是不够的，AI 会用其他 AI 味词替换。必须给**正面锚点**让它对照
3. **缺量化指标**：没有"已经 ≤ 3 处、核心 ≤ 3 处"这类硬阈值，AI 会凭感觉判断"差不多了"

本文件提供三类资产：

- **五段真实公文/咨询报告锚本**（国务院发展研究中心、财政部、国家信息中心、国办、人民日报系）
- **八项文风指纹**（正面对齐 + 负面禁用）
- **AI 味密度指标硬约束**（grep 命令 + 阈值）

## 一、八项文风指纹（正面对齐）

归纳自真实党政公文与国家智库咨询报告，写作时按此对标：

| 编号 | 指纹 | 具体表现 | 示例 |
|---|---|---|---|
| 1 | 段落起手用动宾短语或偏正名词短语 | 不用完整判断句起手 | "坚持统筹规划""推进整合""启动条件总体评估" |
| 2 | 句式以长句＋分号为主 | 一段一气呵成，分号衔接多组平行小句，不刻意切短 | 一段 200—400 字，分号 3—5 个 |
| 3 | 动词偏好收敛 | 公文动词词库较小，反复使用同一批 | 推动／推进／加快／加强／强化／完善／健全／建立／实现／落实／统筹／着力／持续／深入／全面 |
| 4 | 判断词带程度软化 | 避免绝对化判断 | 业已／基本／总体／较为／相对／基本具备／显著提升／初步建立／取得突破性进展 |
| 5 | 结构引入语规范 | 段首常用固定引入短语 | 按照……要求、围绕……、聚焦……、依托……、基于……、通过…… |
| 6 | 连接词规范 | 不用互联网口语 | 同时、此外、在此基础上、进一步、为此、为进一步 |
| 7 | 数字规范 | 年份精确到月 | "2026 年 7 月前""2022 年底前"；层级编号用中文数字（一、（一）1．）；统计数据用阿拉伯数字 |
| 8 | 决策语气软化 | 不用强硬 | 建议／拟／呈请／审议批准／备案／部署／统筹／研究提出；避免"必须""一定要""绝不" |

## 二、AI 味负面清单（严禁出现）

### 句式层

| 模式 | 禁用原因 | 改法 |
|---|---|---|
| "不是……而是……""不只是……而是……""并不是……而是……" | AI 最典型的强对比句，真实公文极少用 | 用"；"分号连接两项平行陈述 |
| "如果说……那么……" | AI 比喻引入句的固定套路 | 改为直接判断式表述 |
| "已经 + 完成动词"密集使用（已经打开／已经打通／已经走通） | 戏剧化判断，AI 爱用 | 改"业已""初步""基本""总体" |
| "这一""这一 X"过度指代 | AI 式代词依赖 | 改"上述""该""此""本" |
| 完整判断句式的一级标题 | 公文标题永远是偏正名词短语 | 改偏正短语（XX的评估／XX与XX） |

### 词汇层

| 禁用词 | 类别 | 替代 |
|---|---|---|
| 倒逼 | 互联网口语 | 推动／促使 |
| 跑通 / 走通 | 互联网口语 | 运行稳定／经实践验证／路径相对稳妥 |
| 踩过坑 | 互联网口语 | 积累实施经验 |
| 错过 | 情绪化 | 若延迟启动／若未能在 XX 之前 |
| 开箱即用 | 科技口语 | 可直接投入使用 |
| 兜底 | 互联网口语 | 保障／基础／依据 |
| 拉通 / 对齐 | 阿里味词 | 统筹／对照／一致 |
| 赋能 | 政策词滥用 | 支持／推动／提升 XX 能力 |
| 抓手 | 阿里/政策词滥用 | 依据／载体／切入点 |
| 核心 / 关键 / 重点（同段 3 次以上） | 程度词堆叠 | 用"主要""总体""整体"分散 |
| 我们 / 大家 / 咱们 | 第一人称 | 公文中去除，改被动或省略 |
| —— （破折号） | AI 最明显指纹 | 一律改逗号或句号重组 |

### AI 过渡词警戒（密集使用即 AI 味）

"总的来看""必须承认""展望未来""值得注意的是""不可忽视的是""毫无疑问""毫不夸张地说""更重要的是""值得一提的是"。出现一两处无妨，密集使用须改写。

## 三、AI 味密度指标硬约束

修改完文稿后，**必须逐项 grep**，超过阈值须回炉：

```bash
FILE="your-file.md"

# 密度指标（阈值内合格）
echo "已经: $(grep -c '已经' $FILE) / 阈值 ≤ 3"
echo "核心: $(grep -c '核心' $FILE) / 阈值 ≤ 3"
echo "这一: $(grep -c '这一' $FILE) / 阈值 ≤ 2"

# 禁用词集合（必须 = 0）
grep -En "兜底|倒逼|跑通|走通|抓手|赋能|拉通|对齐|错过|踩过坑|开箱即用" "$FILE"

# 破折号（必须 = 0）
grep -c "——" "$FILE"

# 强对比句式（必须 = 0）
grep -Ec "不是.{1,10}而是|并不是.{1,10}而是|不只是.{1,10}而是|如果说.{1,5}那么" "$FILE"

# AI 过渡词警戒（不做硬约束，做提示）
grep -En "总的来看|必须承认|展望未来|值得注意的是|不可忽视的是|毫无疑问" "$FILE"
```

若任一硬约束项不通过，须修订重扫，直至全部达标。

## 四、五段真实锚本（对标写作时的"镜子"）

### 锚本一：国务院办公厅《全国一体化政务服务平台移动端建设指南》（2021）

**类型**：纯公文 · 工作原则章节
**特征**：段落起手用"坚持 XX"动宾短语，分号衔接长句

> 坚持统筹规划。强化系统观念，完善标准规范，加强全国一体化平台移动端建设顶层设计，充分发挥国家政务服务平台移动端总枢纽作用，推动各地区各部门移动政务服务资源整合和政务服务平台移动端集约化建设，全面提升一体化服务能力。
>
> 坚持需求引领。聚焦企业和群众办事难点堵点问题，顺应企业和群众新要求、新期盼，积极推动企业和群众经常办理的事项向移动端延伸，提升移动政务服务供给水平，全面优化用户体验，推动政务服务事项从"掌上可办"向"掌上好办"转变。
>
> 坚持安全可控。全面落实总体国家安全观，树牢网络安全底线思维，统筹发展和安全，增强移动政务服务一体化安全防护能力，加强对重要政务数据、敏感个人信息等的保护，确保政务网络和数据信息安全。

**学什么**：

- 每段以"坚持 XX"偏正短语作为段首小标题，然后接展开
- 动词词库限定在"强化、完善、加强、推动、提升、聚焦、顺应、统筹、落实、树牢、增强"等
- 句子较长，用逗号分隔多组动宾短语，不刻意切短

### 锚本二：国家信息中心《我国低空经济发展面临的问题与政策建议》（2024-12）

**类型**：国家智库咨询报告 · 问题诊断章节
**特征**：客观描述现状＋具体数据支撑＋判断句收尾

> 近年来，虽然我国通用机场数量不断增加，地面服务保障设施持续完善，但仍存在通航机场数量较少、地区分布不均衡等问题。截至 2023 年底，不考虑私人机场的情况下，全国在册管理的通用机场数量仅有 449 个，仅为美国公共通航机场数量的 11% 左右；全国通航使用低空空域不足 30%，且分布不均、未能成网连片，特别是重点区域覆盖率不足，难以满足多元化的服务需求。

**学什么**：

- "虽然……但仍存在……"作为问题引入，是咨询报告的标准句式（比"不是……而是……"克制得多）
- 每个判断都有具体数据（449 个、11%、30%）紧随其后
- 结尾用"难以满足多元化的服务需求"这种**克制的判断**，而非"严重制约"等夸张表述

### 锚本三：国家信息中心《我国低空经济发展面临的问题与政策建议》 · 政策建议章节

**类型**：国家智库咨询报告 · 政策建议章节
**特征**：动宾短语标题＋具体动作清单

> **（一）夯实低空经济基础设施建设**
>
> 完善起降基础设施网络体系，加大通用机场建设力度，规划建设一批无人机小型起降平台、中型起降场、大型起降枢纽、eVTOL 起降场、直升机起降平台。建设低空飞行服务站、充换电及公共测试场以及全空间无人系统等相关基础设施，完善路网、电力等基础设施，增补低空航空气象监测设施，强化通用机场、起降场地等平战转换功能。

**学什么**：

- 政策建议章节的标题永远是"夯实 XX""推进 XX""增强 XX""强化 XX""加强 XX"等动宾短语
- 正文几乎不用主语（省略"政府／相关部门")，直接动词开头
- 多组并列动作用顿号和句号分隔，不用破折号或括号

### 锚本四：国务院发展研究中心 钱平凡《数据经济高质量发展的考量与政策建议》（2024）

**类型**：国家智库专家政策建议文章
**特征**：用"一是……二是……"做标准公文列举，判断＋论据＋数据完整呈现

> **一是作为数据经济燃料的数据产量大增长快。** 2023 年，我国数据生产总量达 32.85 泽字节（ZB），同比增长 22.44%。5G、AI、物联网技术的创新发展及智能设备的规模应用，推动数据生产规模快速增长。
>
> **二是为数据经济发展提供支撑的数字基础设施持续稳固并不断扩容提速。** 我国拥有全球最大的 5G 网络，基站总数超过 380 万个，网民规模达到 10.92 亿。全国在用数据中心标准机架超过 810 万架，位居全球第二，算力总规模达 230 EFLOPS（每秒执行的浮点运算次数），近 5 年平均增速近 30%，数据存储力总规模约 1200 EB。

**学什么**：

- "一是 XX 的 XX"是偏正短语判断句，后接数据论证，是咨询报告的标准结构
- 每个"一是／二是"的判断都是一个完整的偏正短语，加粗后起到小标题作用
- 数据支撑**极其具体**：精确到小数位（32.85、22.44%）、年份（2023 年）、机构出处（若有）

### 锚本五：财政部经济建设司《2025 年中小企业数字化转型城市试点工作通知》（2025-05）

**类型**：现役公文 · 基本原则章节
**特征**："（一）围绕 XX，突出 XX"八字动宾短语标题

> **（一）围绕企业需求，突出转型实效。** 精准把握中小企业数字化转型实际需求，剖析"不愿转、不敢转、不会转"深层次原因，靶向施策推动中小企业围绕研发、生产等关键环节开展深度改造。要把中小企业数字化转型作为助企纾困和提升竞争力的关键举措，助力企业在创新、市场、提质、降本、增效、绿色、安全等方面实现价值提升，切实增强企业获得感与竞争力。
>
> **（二）供需双向发力，强化专业供给。** 以中小企业数字化转型需求为导向，重点遴选、培育一批既懂行业又懂数字化的服务商，针对性开发一批行业属性强、赋能效果优的"小快轻准"解决方案，市场化、常态化支撑细分行业中小企业开展高质量数字化转型。

**学什么**：

- "（一）围绕 XX，突出 XX"这种八字对仗动宾短语做小标题，是财政部系统最典型的公文结构
- 正文密集使用"精准把握""靶向施策""重点遴选、培育""针对性开发""市场化、常态化支撑"等四字／六字短语
- 用"要把 XX 作为 XX"这种公文套路表达决策导向，比"XX 非常重要"公文感更强

## 五、写作与修订的四步闭环

使用本锚本做写作或修订时，建议按以下四步走：

**第一步：对标阅读（写作前）**

根据文稿类型选择锚本：

- 工作原则／指导思想段 → 锚本一（国办移动端指南）
- 现状诊断／问题章节 → 锚本二（国家信息中心低空经济）
- 政策建议／对策章节 → 锚本三（同上）或锚本四（钱平凡）
- 基本原则／分条论述 → 锚本五（财政部通知）

通读对应锚本后再动笔，让"镜子"先进入脑中。

**第二步：对标写作（写作中）**

- 每段起手默念锚本的句式：**"坚持 XX""推进 XX""一是 XX""（一）围绕 XX，突出 XX"**
- 判断词默念锚本的软化词：**业已／基本／总体／较为／相对**
- 连接词默念锚本的标配：**同时／此外／在此基础上／进一步／为此**

**第三步：扫描自检（写作后）**

逐项跑第三节的 grep 命令，超阈值即修订。

**第四步：念改（交付前）**

按《怎样写作》建议，建议用户打印后逐段朗读，凡读起来像"AI 总结"的句子继续改。念改时对照锚本，问自己：**"这段话能放进国办文件或国家信息中心报告里吗？"** 不能则继续改。

如 URL 失效，用 Firecrawl 续抓同类新文件（保留本锚本库的结构，仅替换具体段落）：

```bash
firecrawl search "国家信息中心 问题与政策建议" --scrape --limit 3 -o /tmp/new-anchor.json --json
firecrawl search "财政部 实施方案 通知" --scrape --limit 3 -o /tmp/new-mof.json --json
firecrawl search "国务院办公厅 印发 实施方案" --scrape --limit 3 -o /tmp/new-gba.json --json
```

重新续抓的文件建议纳入本文件作为新锚本，保持锚本库的时效性和多样性。
