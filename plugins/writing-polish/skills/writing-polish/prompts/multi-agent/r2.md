# R2 fresh-eye 反查（不传 R1 trajectory）

> Layer 3 第二轮：R1 已经 3-5 视角全扫，R2 派**单个** fresh subagent 反查"R1 之后还能发现什么"。**铁律：不传 R1 trajectory**。

## 为什么不传 R1 trajectory

| 传 R1 输出（错） | 不传 R1（对，本模板做法） |
|---|---|
| subagent 被 R1 既有 finding 锚定，倾向于"加严同一处" | subagent 独立反推 spec，发现 R1 漏掉的新维度 |
| 阶段二沦为 R1 的复读 / 同义改写 | 阶段二真正承担"质量保证"角色 |
| 主对话最终收到 N+M 条 finding 但 80% 重叠 | 收到 N+K 条 finding，K 几乎是全新维度 |

Cognition 2026-04 范式 + Devin 实证（每 PR 多抓 2 个 bug，58% severe）适用此处。

## 主对话调用范式

```
Agent(
  subagent_type="general-purpose",
  description="R2 fresh-eye 反查",
  prompt=<本模板填好 placeholder>
)
```

**单个** subagent，不并行。fresh subagent 入参**只包含**：
- 待审文档绝对路径
- SSOT 红线路径
- 项目豁免清单

**绝对不传**：
- R1 任一 subagent 的 finding JSON
- 主对话已采纳清单
- "请补 R1 漏掉的"这种引导

## R2 任务 prompt 模板

```markdown
# 任务：R2 全文反查（独立审稿）

## 1. 角色

你是 **总编辑级独立审稿人**（10+ 年 {{GENRE}} 经验），clean context，
你**没有**任何前置审稿信息，第一次看本文档。

## 2. 路径

- **待审文档**：`{{ABSOLUTE_FILE_PATH}}`
- **SSOT 红线**：
  - `references/anti-ai-taste-anchors.md` §1（全部红线）
  - `references/constitution.md`（v5 五维 rubric）
  - `references/revision-checklist.md`（何其芳 12 项 + 三维系统审查）
  - `references/logic-and-structure.md`（结构主线）
- **项目豁免**：`memory/project-exemptions/{{PROJECT}}.yaml`

## 3. 维度

**全维度通审**，但你的价值在抓 R1 单视角 subagent 可能漏掉的**跨维度症候**：
- 立意主题（一句话能否概括？跑题没？"全而又全"？）
- 主线清晰度（结构有没有断裂 / 反复）
- 跨章节一致性（前后矛盾 / 术语漂移）
- 数据 / 引用与上下文的张力
- **D2 + D5 组合症候**（既套话又模板感，单维 judge 可能各扣 1 分，组合起来才暴露）
- 客户敏感的二阶问题（R1-C 可能只看显性批评，你看含蓄越界 / 暗讽 / 反讽）

不重复 R1 已经能抓的字面红线（如直引号 / em-dash / 抓手），那些 L1 scan 早已 0 命中。

## 4. 约束

同 r1.md §4：援引 SSOT、不批评、不命令、不引用训练记忆。

特别铁律：**你不知道前几轮发生了什么**。
- 不要写"R1 漏掉了 X"——你看不到 R1
- 不要写"补充上一轮"——没有上一轮
- 就当这是项目第一次审稿

## 5. 输出格式

同 r1.md §5 JSON schema。
multi-dimension finding 允许 `dimension: "cross-D2-D5"` 这种组合标签。

## 6. 输出上限

- finding 条数 **≤ 12**（R2 求精不求多）
- summary ≤ 150 字，可写整体判断（"全文最大风险是 X，建议 Y"）

## 7. Clean Context 铁律

再次强调：你**没有** R1 上下文，也**没有**主对话上下文。
你的任务是从 §2 SSOT 出发，独立审一遍。
独立判断本身就是 R2 的价值。
```

## 派 R2 后主对话的下一步

主对话收到 R2 JSON 后：
1. 与 R1 整合 finding 列表（dedup by `line_range + dimension`）
2. 进入 `orchestrator.md` synthesis 阶段（P0-P5 排序 + 决策三问 + Edit 串行落地）
