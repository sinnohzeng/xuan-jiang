# RFC：writing-polish v5.0 LLM-as-judge 混合架构

**状态**：DRAFT
**作者**：xuan-jiang 维护者
**创建**：2026-05-08
**目标版本**：v5.0
**前置版本**：v4.3（上下文感知白名单 + 动态密度阈值）

---

## 一、动机

v4.2 / v4.3 全部基于硬正则匹配 + 上下文白名单。这种范式有 6 个根本盲点：

1. **同形异义无法识别**：防火墙（IT 实物 vs 隐喻战斗）、对标（党政 vs 大厂）、闭环（控制论术语 vs AI 黑话）—— v4.3 的上下文白名单是补丁，不是根治
2. **新词演化滞后**：DeepSeek 改写出的"走深走实""向纵深推进""持续向好"等新型 AI 痕迹不在词典里就漏检
3. **整体文风感缺失**：单句不踩词典但全文"散文 AI 体"依然漏掉（句长方差 + 三段式只是粗代理）
4. **粗颗粒度判定**：整段 PASS / FAIL 而非 dimension-wise scoring，无法定位"这一段在隐喻强度上 2/3"
5. **无 calibration**：当前 evals 是脚本回归测试，无"人类金标准 correlation ≥ 0.85"校准
6. **同形近义变体逃逸**：「赋能」抓住，「赋予能力」「增能」「使能」逃逸；「打造」抓住，「构建」「营造」「锻造」靠橙线

2026 mid-year 业界已演化到混合范式（参见 [Anthropic Engineering "Demystifying evals for AI agents" 2026-01](https://www.anthropic.com/engineering/demystifying-evals-for-ai-agents) 与 [Openlayer "LLM as Judge Guide" 2026-03](https://www.openlayer.com/blog/post/llm-as-judge-evaluation-guide)）：硬规则跑 fast path（deterministic 信号），LLM-as-judge 跑 slow path（subjective dimensions）。

## 二、核心架构

### 2.1 双层执行模型

```
┌─────────────────────────────────────────────┐
│  scan-ai-taste.sh                           │
│                                             │
│  Layer 1 (Fast / 硬规则，< 1s)：            │
│   D1 标点 / 工具残留（破折号 / oaicite）    │
│   D2 显式 AI 套话（赋能 / 综上所述）        │
│   退出码 1 = 任一 Layer 1 红线 FAIL         │
│                                             │
│  Layer 2 (Slow / LLM judge，5-30s)：        │
│   仅在 --llm-judge flag 开启时触发          │
│   D3 隐喻强度（防火墙 IT vs 战斗化）        │
│   D4 大厂 vs 党政语境（对标 / 闭环）        │
│   D5 整体散文 AI 体（句长方差合格但假大空） │
│   退出码 2 = Layer 1 PASS + Layer 2 WARN    │
│   退出码 4 = Layer 2 FAIL（需新增退出码）   │
└─────────────────────────────────────────────┘
```

### 2.2 Rubric Decomposition（5 个 dimension）

| ID | 维度 | 硬规则可处理？ | LLM judge 必要？ |
|---|---|---|---|
| D1 | 标点 / 工具残留（破折号 / oaicite / placeholder dates） | ✅ 完全 | ❌ 不需 |
| D2 | 显式 AI 套话（赋能 / 综上所述 / 让我们一起） | ✅ 80% | 仅近义变体 |
| D3 | 隐喻强度（防火墙 / 闸门 / 跑通 是否在战斗语境） | ❌ 硬规则糊不了 | ✅ 必须 |
| D4 | 大厂 vs 党政语境（对标 / 闭环 / 抓手） | ❌ 硬规则糊不了 | ✅ 必须 |
| D5 | 整体散文 AI 体（句长方差合格但仍假大空） | ❌ 硬规则糊不了 | ✅ 必须 |

每个 dimension 0-3 分 + Unknown：

- 0 = 完全不踩
- 1 = 轻度（一处可疑但语境合理）
- 2 = 中度（多处叠加 / 上下文不合理）
- 3 = 重度（明显套话）
- Unknown = 信息不足无法判断（escape hatch，符合 Anthropic 2026-01 推荐）

## 三、Prompt 范式

```text
你是中文公文与咨询报告写作评审专家。判断下列文段在 [D3 隐喻强度] 这一维度的 AI 味浓度。

文段：{{ paragraph }}

打分标准（0-3）：
0 = 完全无 AI 隐喻战斗化
1 = 轻度（一处可疑但语境合理）
2 = 中度（多处叠加 / 上下文不合理）
3 = 重度（明显套话）
Unknown = 信息不足无法判断

few-shot 示例（cicpa 治理后金标准）：
- "机房部署防火墙 2 台" → 0（IT 实物术语）
- "三层防御加多重防火墙保护核心数据" → 3（隐喻战斗化）
- "对标先进经验" → 0（党政标准用法）
- "我们对标 KPI、内部赛马" → 3（大厂黑话）
- "建设方案对照同业案例形成启示" → 0（咨询报告标准用法）

按 JSON 返回：{"score": 0|1|2|3|"Unknown", "reason": "..."}
```

### 3.1 Bias 校正（Openlayer 2026 范式）

- **pass^k 稳定性**：每段跑 3 次取多数票，rotate few-shot 顺序防 position bias
- **Few-shot 多样化**：每次随机抽取 3-5 条 few-shot 示例（pool size ≥ 20）

### 3.2 模型选择

- **首选**：Anthropic Haiku 4.5（中文性价比最高，speed/cost/accuracy 平衡）
- **BYOK 选项**：DeepSeek（中文圈本地化），用户可在 config 中切换
- **未来探索**：fine-tuned 小模型（< 1B）替代 Haiku 降本

## 四、Calibration Set

### 4.1 数据来源

- **正样本**（应 PASS，得分 0-1）：cicpa 053 治理后稳定段 30-50 段（人类金标准）
- **负样本**（应 FAIL，得分 2-3）：cicpa 053 治理前 AI 浓度高段 30-50 段（人类金标准）
- 后续可从用户的治理 commit 自动抓取（v5.2 self-loopback 数据闭环）

### 4.2 验收标准

- **Correlation ≥ 0.85**（与 Openlayer 2026 推荐一致）
- 不达标时迭代 prompt 至达标，否则不发版

### 4.3 评估流程

```bash
# 跑 calibration 测试
bash evals/calibration-runner.sh --model haiku-4.5 --rounds 3
# 输出：correlation 系数 + 每段得分 + 与人类标注的 disagreement 列表
```

## 五、成本估算

以 cicpa WS1/WS3 完整版 5+7 万字为例：

- 分段 200 字 / 段 → 约 600 段
- 5 个 dimension × 3 次重复 × 600 段 ≈ **9000 次 Haiku 调用**
- 输入 token ≈ 30 万（包含 prompt 模板 + few-shot + 文段）
- 输出 token ≈ 5 万（结构化 JSON）
- 单价：Haiku 4.5 输入 ~$0.80 / 1M tokens、输出 ~$4 / 1M tokens
- **总成本 ≈ $0.5-1.0 / 次完整扫描**

可接受。若用户跑频次高，可加 prompt caching 把成本压到 $0.2-0.4。

## 六、向后兼容

v5.0 默认行为 **不变**：

- `bash scan-ai-taste.sh <file>` 仍只跑 Layer 1 硬规则（与 v4.3 一致）
- 退出码 0 / 1 / 2 / 3 语义保留

opt-in 触发 LLM judge：

- `bash scan-ai-taste.sh <file> --llm-judge` 启用 Layer 2
- 新增退出码 4 = LLM judge FAIL（v5.0 引入）

evals.json 顶层新增 `llm_judge_calibration` 数组，与 `tests` / `regression_fixtures` 三轨并行。

## 八、Out of scope（v5.0 不做）

- ✗ 训练自家 fine-tuned 模型（v6.0+）

- ✗ 跨语言扩展（英文 LLM judge）
- ✗ 替代硬规则（硬规则的速度 / 透明度优势保留）
