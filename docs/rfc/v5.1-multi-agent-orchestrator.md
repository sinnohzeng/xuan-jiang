## 1 · 问题陈述（v5.0 偏离行业最佳实践 3 处）

经 Firecrawl 实测调研 5 个权威来源（2026-05-27 firecrawl quota 内 8 次 search/scrape）：
- Anthropic Skills 官方 docs + best-practices
- Anthropic Multi-Agent Research System Blog (2025-06)

- Pydantic Evals LLM-as-Judge (2026-02)

v5.0.0 偏离 3 处：

### 偏离 1：第 3 层 placeholder 模板违背"主智能体判断"

`prompts/multi-agent/{r1,r2,pre-mod,orchestrator,_task-spec-skeleton}.md` 5 个文件共 499 行，全部带 `{{VIEW_ID}}` `{{ROLE_TITLE}}` `{{ABSOLUTE_FILE_PATH}}` `{{GENRE}}` `{{MAX_FINDINGS}}` 等 7+ 占位符。主对话需手工 copy-paste + fill placeholder + 调 Agent 工具——**机械填空，不是智能判断**。

**业界共识**（Anthropic Multi-Agent System 2025-06 论文核心精神）："Teach orchestrator how to delegate"——orchestrator 应根据任务特征自主决定 worker 数 / 视角 / 收敛阈值。模板填空把主智能体降级为执行机器人。

### 偏离 2：Cohen's κ=0.368 vs 0.80 目标

v5.0-rc1 calibration 实测：
- D2/D3 κ=1.0 完美
- D1/D4/D5 κ=0.0
- D5 accuracy 74.5%（judge 总输出"无违规"被 prompt 固化）
- D4 复盘 / 拉通 等大厂内训词侵入 G8 咨询语境未抓住

**根因**：constitution.md §5 仅 6 个 few-shot，D4 D5 各只 1 例；§6 模糊副词雷达存在但未在 judge prompt inline；§2.8 G8 红线没有 "大厂内训词侵入" 这一条。

### 偏离 3：SKILL.md 344 行违背 Anthropic Progressive Disclosure

Anthropic 官方 best-practices: Metadata (~100 tokens) → SKILL.md (<5k tokens) → Resources/Code（无上限）。344 行混杂 DOCX 处理 + 写作方法论 + 三层架构 + 修改哲学 + 文体路由，与已有 11 个 references/ 文件大量重复（如 §2 DOCX 重复 docx-editing-guide.md 340 行）。

**业界共识**（Tessl Registry 2026-03 + Anthropic Skills cookbook）：SKILL.md 是**路由 + 决策树**，知识全部下沉 references/。

## 2 · 设计原则（用户 2026-05-27 三轮明确）

1. **没有历史包袱**：可以大刀阔斧重构，不需要兼容动作
2. **主智能体判断决策**：禁止脚本化 placeholder 填空，禁止 Python 编排器
3. **遵循 Anthropic 最新最佳实践**：Multi-Agent System 2025-06 + Skills best-practices + Progressive Disclosure
4. **模型解耦原则保留**：主对话即 orchestrator，不调外部 API
5. **多路子智能体本次实装**：v5.1 必含端到端实战，主对话 Opus 4.7 1M = lead，派 1 路 Opus 4.7 1M Pre-mod/R2 + 多路 Sonnet 4.6 1M R1
6. **不设测试 release gate**：灰度上线 + 实际使用中迭代
7. **DDD SSOT 文档同步**：编码同时同步所有受影响文档，零漂移
